# Refactor R3 : 관리자 콘솔 도구

## 1. 목적

DB에 저장된 데이터의 현재 상태를 **메인 앱과 독립적으로** 실시간 조회할 수 있는 관리자 CLI 도구를 제공한다.  
R2(DB 연동) 완료 후 사용 가능하다.

---

## 2. 실행 방법

```bash
.venv/Scripts/python.exe tools/admin_console.py
```

---

## 3. 화면 구성

### 3.1 시작 화면

```
════════════════════════════════════════════════════════════
  반도체 시료 생산주문관리 — 관리자 콘솔
  DB: data/semi.db  |  연결됨
════════════════════════════════════════════════════════════
명령어를 입력하세요. (help: 도움말, exit: 종료)
admin >> 
```

---

### 3.2 지원 명령어

| 명령어 | 설명 | 예시 |
|--------|------|------|
| `samples` | 전체 시료 목록 조회 | `samples` |
| `orders` | 전체 주문 목록 조회 | `orders` |
| `orders <STATUS>` | 상태별 주문 필터 | `orders RESERVED` |
| `order <ID>` | 주문 단건 상세 조회 | `order ORD-20260508-001` |
| `queue` | 생산 큐 전체 조회 | `queue` |
| `stats` | 전체 현황 요약 | `stats` |
| `history <N>` | 최근 N건 주문 이력 | `history 10` |
| `help` | 명령어 목록 | `help` |
| `exit` | 종료 | `exit` |

## 4. 아키텍처

```
tools/admin_console.py
    │
    ├─ DBConnection.get()          ← R2 DB 연결 재사용
    │
    ├─ AdminCommandParser          ← 명령어 파싱
    │
    └─ AdminQueryService           ← 조회 전용 서비스
        ├─ SampleRepository.find_all()
        ├─ OrderRepository.find_all() / find_by_status()
        ├─ ProductionRepository.find_by_status()
        └─ OrderRepository.get_recent_history()
```

> 관리자 콘솔은 **읽기 전용(Read-Only)**이다. 데이터 변경 명령은 제공하지 않는다.
