## 2. 기능 상세

### 2.1 출고 대기 목록

- `CONFIRMED` 상태 주문만 표시

```
──────────────────────────────────────────────────────────────────────
  출고 대기 목록 (총 4건)
──────────────────────────────────────────────────────────────────────
 NO   주문 ID              고객명              시료명         수량  승인일시
──────────────────────────────────────────────────────────────────────
  1   ORD-20260507-002    XYZ소재             인화인듐 박막     5   05-07 09:12
  2   ORD-20260508-001    (주)반도체코리아    실리콘 기판       30   05-08 10:25
  3   ORD-20260508-002    ABC전자             인화인듐 박막    20   05-08 14:55
  4   ORD-20260508-005    테스트고객          갈륨비소 웨이퍼  10   05-08 15:10
──────────────────────────────────────────────────────────────────────
```

- 정렬 : 승인일시 오름차순 (오래된 순 우선)
- `NO` : 일괄 출고 시 선택 번호로 사용

---

### 2.2 단건 출고 처리

#### 입력 흐름

```
──────────────────────────────────────
  단건 출고 처리
──────────────────────────────────────
주문 ID 입력 >> ORD-20260508-001

──────────────────────────────────────
  출고 처리 확인
──────────────────────────────────────
  주문 ID    : ORD-20260508-001
  고객명     : (주)반도체코리아
  시료       : A-002 실리콘 기판
  출고 수량  : 30 개
  현재 상태  : CONFIRMED
──────────────────────────────────────
출고 처리하시겠습니까? (Y/N) >> Y

[완료] 출고 처리 완료.
       주문 ID  : ORD-20260508-001
       상태     : CONFIRMED → RELEASE
       출고일시 : 2026-05-08 15:45:22
```

#### 오류 케이스

| 오류 | 메시지 |
|------|--------|
| 존재하지 않는 주문 | `[오류] 해당 주문을 찾을 수 없습니다.` |
| CONFIRMED 아닌 주문 | `[오류] 출고 가능한 상태가 아닙니다. (현재: RESERVED)` |

---

### 2.3 일괄 출고 처리

#### 입력 흐름

```
──────────────────────────────────────────────────────────────────────
  출고 대기 목록 (총 4건)
──────────────────────────────────────────────────────────────────────
 NO   주문 ID              고객명              시료명         수량
──────────────────────────────────────────────────────────────────────
  1   ORD-20260507-002    XYZ소재             인화인듐 박막     5
  2   ORD-20260508-001    (주)반도체코리아    실리콘 기판       30
  3   ORD-20260508-002    ABC전자             인화인듐 박막    20
  4   ORD-20260508-005    테스트고객          갈륨비소 웨이퍼  10
──────────────────────────────────────────────────────────────────────
출고할 NO 입력 (쉼표 구분, 전체: all) >> 1,2,4

──────────────────────────────────────
  일괄 출고 확인 (3건)
──────────────────────────────────────
  - ORD-20260507-002  XYZ소재  인화인듐 박막  5개
  - ORD-20260508-001  (주)반도체코리아  실리콘 기판  30개
  - ORD-20260508-005  테스트고객  갈륨비소 웨이퍼  10개
──────────────────────────────────────
일괄 출고 처리하시겠습니까? (Y/N) >> Y

[완료] 3건 출고 처리 완료. (2026-05-08 15:46:01)
  ORD-20260507-002 → RELEASE
  ORD-20260508-001 → RELEASE
  ORD-20260508-005 → RELEASE
```

#### 일괄 처리 규칙

| 항목 | 내용 |
|------|------|
| `all` 입력 | 전체 대기 목록 일괄 출고 |
| 번호 선택 | 쉼표(`,`) 구분 입력 |
| 중복 번호 | 중복 제거 후 처리 |
| 잘못된 번호 | `[오류] 유효하지 않은 NO: 9. 건너뜁니다.` |

## 3. 출고 처리 로직

```
출고 처리 흐름:
    1. 주문 상태 확인 (CONFIRMED 아니면 오류)
    2. 주문 상태 → RELEASE
    3. 출고일시 기록 (released_at)
    ※ 재고는 승인 시 이미 차감되었으므로 추가 변경 없음
```

## 5. 데이터 흐름

```
release_menu.py (화면 입출력)
    │
    ▼
release_service
    ├─ get_pending_list()
    │   └─ order_repo.find_by_status(CONFIRMED)
    │      .orderBy(updated_at ASC)
    │
    ├─ release_one(order_id)
    │   ├─ order_repo.get(order_id)
    │   └─ order_repo.update(status=RELEASE, released_at=now)
    │
    └─ release_bulk(order_ids: list[str])
        └─ for each order_id: release_one(order_id)
```
