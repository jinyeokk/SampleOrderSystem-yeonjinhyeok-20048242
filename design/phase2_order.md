## 2. 기능 상세

### 2.1 주문 접수

#### 입력 흐름

```
──────────────────────────────────────
  주문 접수
──────────────────────────────────────
고객명       >> (주)반도체코리아

──────────────────────────────────────
  등록된 시료 목록
──────────────────────────────────────
 ID        이름                  재고
──────────────────────────────────────
 A-001     갈륨비소 웨이퍼        120
 A-002     실리콘 기판              45
 B-001     인화인듐 박막             0
──────────────────────────────────────
시료 ID     >> A-002
주문 수량   >> 30

──────────────────────────────────────
  주문 내용 확인
──────────────────────────────────────
  주문 ID   : ORD-20260508-001
  고객명    : (주)반도체코리아
  시료      : A-002 실리콘 기판
  주문수량  : 30 개
  현재재고  : 45 개
──────────────────────────────────────
접수하시겠습니까? (Y/N) >> Y

[완료] 주문이 접수되었습니다. (주문 ID: ORD-20260508-001, 상태: RESERVED)
```

#### 주문 ID 생성 규칙

```
ORD-{YYYYMMDD}-{일련번호 3자리}
예) ORD-20260508-001
```

#### 입력 필드 및 검증 규칙

| 필드 | 검증 규칙 |
|------|-----------|
| 고객명 | 필수, 최대 50자 |
| 시료 ID | 필수, 등록된 시료만 허용 |
| 주문 수량 | 필수, 1 이상 정수 |

#### 재고 사전 안내

- 재고가 주문 수량보다 부족한 경우 접수 전 안내 메시지 표시

```
[안내] 현재 재고(0개)가 주문 수량(30개)보다 부족합니다.
       승인 시 생산 라인으로 전환됩니다.
접수하시겠습니까? (Y/N) >>
```

### 2.2 주문 목록 조회

#### 화면 출력

```
──────────────────────────────────────────────────────────────────────
  주문 목록 (총 5건)
──────────────────────────────────────────────────────────────────────
 필터: 전체  |  1.RESERVED  2.PRODUCING  3.CONFIRMED  4.RELEASE  5.REJECTED
 상태 필터 선택 (Enter: 전체) >> 
──────────────────────────────────────────────────────────────────────
 주문 ID              고객명              시료명         수량  상태
──────────────────────────────────────────────────────────────────────
 ORD-20260508-001    (주)반도체코리아    실리콘 기판      30   RESERVED
 ORD-20260507-003    ABC전자             갈륨비소 웨이퍼  10   PRODUCING
 ORD-20260507-002    XYZ소재             인화인듐 박막     5   CONFIRMED
 ORD-20260506-001    (주)반도체코리아    실리콘 기판      20   RELEASE
 ORD-20260505-002    테스트고객          갈륨비소 웨이퍼   3   REJECTED
──────────────────────────────────────────────────────────────────────
```

#### 상태 필터

- 기본값 : 전체 표시
- 번호 선택 시 해당 상태 주문만 필터링
- `REJECTED` 는 필터 선택 시에만 표시 (기본 전체 목록에 포함)

#### 정렬 기준

- 기본 정렬 : 접수일시 내림차순 (최신 순)

## 4. 모델 정의

```python
from enum import Enum
from dataclasses import dataclass, field
from datetime import datetime

class OrderStatus(Enum):
    RESERVED  = "RESERVED"
    REJECTED  = "REJECTED"
    PRODUCING = "PRODUCING"
    CONFIRMED = "CONFIRMED"
    RELEASE   = "RELEASE"

@dataclass
class Order:
    order_id: str
    customer_name: str
    sample_id: str
    quantity: int
    status: OrderStatus = OrderStatus.RESERVED
    reject_reason: str = ""
    created_at: datetime = field(default_factory=datetime.now)
    updated_at: datetime = field(default_factory=datetime.now)
```
