# Phase 2 : 시료 주문

## 1. 개요

고객이 원하는 시료와 수량을 요청하면 **주문 담당자**가 시스템에 주문을 생성한다.  
메인 메뉴에서 **[2] 시료 주문** 선택 시 서브메뉴 없이 주문 화면으로 바로 진입한다. 시료 ID 입력 후 별도 피드백 없이 고객명·수량을 연속 입력하며, 재고 확인은 [3] 승인 메뉴에서 처리한다.  
주문 생성 시점의 주문 상태는 항상 `RESERVED`이다.

---

## 2. 화면 흐름

```
메인 메뉴
    │
    └── [2] 시료 주문 ──────────────────────────────────────┐
                                                            ▼
                                                   시료 주문 화면
                                                   (시료 ID → 고객명 → 수량)
                                                            │
                                         Y.예약접수 ◄─ 선택 ─► N.취소
                                             │                    │
                                        완료 화면            메인 메뉴로
                                   (주문 번호·상태·안내)         복귀
                                             │
                                        메인 메뉴로 복귀
```

## 3. 기능 상세

### 3.1 시료 주문

#### 입력 순서

```
시료 ID → 고객명 → 주문 수량
```

#### 입력 흐름

```
────────────────────────────────────────────────────────────
  시료 주문
────────────────────────────────────────────────────────────
시료 ID     >> A-002
고객명      >> (주)반도체코리아
주문 수량   >> 30

────────────────────────────────────────────────────────────
  입력 내용 확인
────────────────────────────────────────────────────────────
  시료      : 실리콘 기판 (A-002)
  고객명    : (주)반도체코리아
  주문 수량 : 30 ea
────────────────────────────────────────────────────────────
  Y. 예약 접수
  N. 취소
선택 >> Y

[완료] 예약 접수 완료.
────────────────────────────────────────────────────────────
  주문 번호 : ORD-20260508-001
  현재 상태 : RESERVED
────────────────────────────────────────────────────────────
[안내] 재고 확인은 [3] 승인 메뉴에서 직접 진행하세요.
```

#### 주문 취소

```
  Y. 예약 접수
  N. 취소
선택 >> N
[안내] 주문이 취소되었습니다.
```

취소 시 메인 메뉴로 복귀한다.

#### 입력 필드 및 검증 규칙

| 필드 | 검증 규칙 |
|------|-----------|
| 시료 ID | 필수, 등록된 시료만 허용 — 없으면 즉시 오류 후 메인 복귀 |
| 고객명 | 필수, 최대 50자 |
| 주문 수량 | 필수, 1 이상 정수 |

#### 시료 ID 오류

```
시료 ID     >> X-999
[오류] 등록되지 않은 시료 ID입니다: X-999
```

#### 주문 ID 생성 규칙

```
ORD-{YYYYMMDD}-{일련번호 3자리}
예) ORD-20260508-001
```

## 4. 데이터 흐름

```
메인 메뉴 [2] 선택
    │
    ▼
order_menu.run(ctx)  ← 서브메뉴 없이 바로 주문 화면 진입
    ├─ 시료 ID 입력 → sample_service.get(sample_id) → 시료 존재 확인만 수행
    ├─ 고객명 입력
    ├─ 주문 수량 입력
    └─ Y.예약접수 / N.취소 선택
          │
          ▼
order_service.place_order(customer_name, sample_id, quantity)
    ├─ 주문 ID 자동 생성
    └─ 상태 RESERVED로 저장
```

## 5. 모델 정의

```python
class OrderStatus(Enum):
    RESERVED  = "RESERVED"   # 주문 완료, 승인 대기
    REJECTED  = "REJECTED"   # 거절됨
    PRODUCING = "PRODUCING"  # 생산 진행 중
    CONFIRMED = "CONFIRMED"  # 출고 대기
    RELEASE   = "RELEASE"    # 출고 완료

@dataclass
class Order:
    order_id: str
    customer_name: str
    sample_id: str
    quantity: int
    status: OrderStatus = OrderStatus.RESERVED
    reject_reason: str = ""
    created_at: datetime = field(default_factory=datetime.now)
    updated_at: datetime = field(default_factory=datetime.now)
    released_at: Optional[datetime] = None
```
