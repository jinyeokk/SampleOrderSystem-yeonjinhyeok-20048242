# Phase 2 : 시료 주문

## 1. 개요

고객이 원하는 시료와 수량을 요청하면 **주문 담당자**가 시스템에 예약을 생성한다.  
예약 생성 시점의 주문 상태는 항상 `RESERVED`이며, 이후 승인·거절·생산은 Phase 3에서 처리한다.

## 3. 기능 상세

### 3.1 시료 예약

주문 담당자가 고객의 요청을 받아 시료 예약을 생성한다.

#### 입력 순서

```
시료 ID → 고객명 → 주문 수량
```

#### 입력 흐름 (재고 충분)

```
────────────────────────────────────────────────────────────
  시료 예약
────────────────────────────────────────────────────────────
시료 ID     >> A-002

  시료명    : 실리콘 기판
  현재 재고 : 45 ea

고객명      >> (주)반도체코리아
주문 수량   >> 30

────────────────────────────────────────────────────────────
  예약 내용 확인
────────────────────────────────────────────────────────────
  주문 ID   : ORD-20260508-001
  시료      : A-002 실리콘 기판
  고객명    : (주)반도체코리아
  주문 수량 : 30 ea
  현재 재고 : 45 ea
────────────────────────────────────────────────────────────
예약하시겠습니까? (Y/N) >> Y

[완료] 예약이 완료되었습니다. (주문 ID: ORD-20260508-001, 상태: RESERVED)
```

#### 입력 흐름 (재고 부족)

시료 ID 입력 직후 재고 상태를 표시하고, 최종 확인 전 생산 전환 안내를 출력한다.

```
────────────────────────────────────────────────────────────
  시료 예약
────────────────────────────────────────────────────────────
시료 ID     >> B-001

  시료명    : 인화인듐 박막
  현재 재고 : 0 ea ⚠

고객명      >> ABC전자
주문 수량   >> 20

[안내] 현재 재고(0 ea)가 주문 수량(20 ea)보다 부족합니다.
       승인 시 생산 라인으로 전환됩니다.

────────────────────────────────────────────────────────────
  예약 내용 확인
────────────────────────────────────────────────────────────
  주문 ID   : ORD-20260508-002
  시료      : B-001 인화인듐 박막
  고객명    : ABC전자
  주문 수량 : 20 ea
  현재 재고 : 0 ea ⚠
────────────────────────────────────────────────────────────
예약하시겠습니까? (Y/N) >> Y

[완료] 예약이 완료되었습니다. (주문 ID: ORD-20260508-002, 상태: RESERVED)
```

#### 입력 필드 및 검증 규칙

| 필드 | 검증 규칙 |
|------|-----------|
| 시료 ID | 필수, 등록된 시료만 허용 (없으면 즉시 오류) |
| 고객명 | 필수, 최대 50자 |
| 주문 수량 | 필수, 1 이상 정수 |

#### 시료 ID 오류

존재하지 않는 ID를 입력한 경우 재입력 없이 즉시 상위 메뉴로 복귀한다.

```
시료 ID     >> X-999
[오류] 등록되지 않은 시료 ID입니다: X-999
```

#### 주문 ID 생성 규칙

```
ORD-{YYYYMMDD}-{일련번호 3자리}
예) ORD-20260508-001
```

### 3.2 주문 목록 조회

전체 주문 이력을 조회하며, 상태 필터로 원하는 상태만 볼 수 있다.

#### 화면 출력

```
────────────────────────────────────────────────────────────
  주문 목록 (총 5건)
────────────────────────────────────────────────────────────
  필터: 전체 | 1.RESERVED  2.PRODUCING  3.CONFIRMED  4.RELEASE  5.REJECTED
  상태 필터 선택 (Enter: 전체) >> 
────────────────────────────────────────────────────────────
 주문 ID              고객명              시료명        수량  상태
────────────────────────────────────────────────────────────
 ORD-20260508-002    ABC전자             인화인듐 박막   20   RESERVED
 ORD-20260508-001    (주)반도체코리아    실리콘 기판     30   RESERVED
 ORD-20260507-003    ABC전자             갈륨비소 웨이퍼 10   PRODUCING
 ORD-20260507-002    XYZ소재             인화인듐 박막    5   CONFIRMED
 ORD-20260506-001    (주)반도체코리아    실리콘 기판     20   RELEASE
────────────────────────────────────────────────────────────
```

#### 표시 규칙

| 항목 | 내용 |
|------|------|
| 기본 정렬 | 예약일시 내림차순 (최신 순) |
| 기본 필터 | 전체 (REJECTED 포함) |
| REJECTED 표시 | 필터 선택 시에만 별도 조회 가능 |

```
  주문 수량 : 30 ea
  예약일시  : 2026-05-08 14:23:11
  최종 변경 : 2026-05-08 14:23:11
  거절 사유 : -
────────────────────────────────────────────────────────────
```

## 4. 데이터 흐름

```
주문 담당자 입력 (시료 ID → 고객명 → 주문 수량)
    │
    ▼
order_menu.py
    ├─ 시료 ID 입력 → sample_service.get(sample_id) → 시료 정보 즉시 표시
    ├─ 재고 < 주문 수량 → 생산 전환 안내 표시
    └─ 예약 확인 (Y/N)
    │
    ▼
order_service.place_order(customer_name, sample_id, quantity)
    ├─ 주문 ID 자동 생성 (ORD-{날짜}-{순번})
    └─ 상태 RESERVED로 저장
    │
    ▼
order_repo.save(order)
```

## 5. 모델 정의

```python
class OrderStatus(Enum):
    RESERVED  = "RESERVED"   # 예약 완료, 승인 대기
    REJECTED  = "REJECTED"   # 거절됨
    PRODUCING = "PRODUCING"  # 생산 진행 중
    CONFIRMED = "CONFIRMED"  # 출고 대기
    RELEASE   = "RELEASE"    # 출고 완료

@dataclass
class Order:
    order_id: str
    customer_name: str
    sample_id: str
    quantity: int
    status: OrderStatus = OrderStatus.RESERVED
    reject_reason: str = ""
    created_at: datetime = field(default_factory=datetime.now)
    updated_at: datetime = field(default_factory=datetime.now)
    released_at: Optional[datetime] = None
```
