# Phase 0 : 메인 화면

## 1. 목적

애플리케이션 진입점과 메인 메뉴를 구성한다.  
이후 모든 Phase의 서브메뉴는 이 메인 화면을 통해 진입하므로, Phase 0은 **전체 앱의 뼈대(shell)** 역할을 한다.  
서브메뉴에서 돌아올 때마다 **현황 요약 패널**이 갱신되어, 담당자가 메뉴 선택 전 주요 수치를 한눈에 파악할 수 있다.

---

## 2. 화면 흐름

```
[앱 시작]
    │
    ▼
시작 배너 출력 (최초 1회)
    │
    ▼
┌──────────────────────────────────────┐
│  현황 요약 패널 갱신                  │◄─────────────────────┐
│  (등록 시료 / 총 재고 / 전체 주문 /  │                      │
│   생산 대기 / 주문 상태 요약)         │                      │
└──────────────────────────────────────┘                      │
    │                                                         │
    ▼                                                         │
┌──────────────────────────────────────┐                      │
│            메인 메뉴                  │                      │
└──────────────────────────────────────┘                      │
    │                                                         │
    ├── 1~6 선택 → 서브메뉴 실행 ──────────── 종료 후 복귀 ───┘
    │
    └── 0 선택 → 종료 확인 → Y → [앱 종료]
                           → N → 현황 갱신 후 메뉴 재표시
```

> **핵심**: 현황 요약 패널은 메인 메뉴가 표시될 때마다 **매번 새로 계산**한다.  
> 서브메뉴 작업(주문 접수, 승인 등) 결과가 즉시 반영된다.

---

## 3. 화면 정의

### 3.1 시작 배너

앱 최초 실행 시 한 번만 출력한다.

```
════════════════════════════════════════════════════════════
  반도체 시료 생산주문관리 시스템 v1.0
  초기화 완료. 시스템이 준비되었습니다.
════════════════════════════════════════════════════════════
```

---

### 3.2 현황 요약 패널

메인 메뉴가 표시될 때마다 최신 데이터로 갱신되어 메뉴 바로 위에 출력된다.

```
────────────────────────────────────────────────────────────
  현황 요약                            2026-05-08 15:30:00
────────────────────────────────────────────────────────────
   등록 시료      총 재고      전체 주문      생산 대기
  ──────────    ──────────    ──────────    ──────────
      3종          165개          8건            3건
────────────────────────────────────────────────────────────
  RESERVED: 2건   PRODUCING: 2건   CONFIRMED: 3건
────────────────────────────────────────────────────────────
```

#### 지표 정의

| 지표 | 데이터 출처 | 설명 |
|------|-------------|------|
| 등록 시료 | `SampleRepository.find_all()` | 전체 시료 종 수 (`N종`) |
| 총 재고 | `sum(s.stock for s in samples)` | 전체 시료 재고 합계 (`N개`) |
| 전체 주문 | 주문 수 (REJECTED 제외) | RESERVED + PRODUCING + CONFIRMED + RELEASE 합계 (`N건`) |
| 생산 대기 | `ProductionRepository.find_by_status(WAITING)` | 생산 큐 대기 건수 (`N건`) |

#### 주문 상태 요약 (2행)

| 상태 | 의미 | 표시 조건 |
|------|------|-----------|
| `RESERVED` | 승인 대기 | 항상 표시 |
| `PRODUCING` | 생산 진행 중 | 항상 표시 |
| `CONFIRMED` | 출고 대기 | 항상 표시 |

> RELEASE는 완료 건이므로 운영 현황과 무관하여 2행에서 생략한다.  
> REJECTED는 시스템 전체에서 집계 제외 원칙을 따른다.

#### 빈 상태 (데이터 없음)

```
────────────────────────────────────────────────────────────
  현황 요약                            2026-05-08 09:00:00
────────────────────────────────────────────────────────────
   등록 시료      총 재고      전체 주문      생산 대기
  ──────────    ──────────    ──────────    ──────────
      0종            0개          0건            0건
────────────────────────────────────────────────────────────
  RESERVED: 0건   PRODUCING: 0건   CONFIRMED: 0건
────────────────────────────────────────────────────────────
```

### 3.3 메인 메뉴

```
╔══════════════════════════════════════╗
║   반도체 시료 생산주문관리 시스템    ║
╠══════════════════════════════════════╣
║  1. 시료 관리                        ║
║  2. 시료 주문                        ║
║  3. 주문 승인/거절                   ║
║  4. 모니터링                         ║
║  5. 생산라인 조회                    ║
║  6. 출고처리                         ║
║  0. 종료                             ║
╚══════════════════════════════════════╝
메뉴 선택 >> 
```

---

### 3.4 잘못된 입력

```
메뉴 선택 >> 9
[오류] 유효하지 않은 선택입니다.
```

메시지 출력 후 현황 패널과 메뉴를 즉시 다시 표시한다.

---

### 3.5 종료 확인

```
메뉴 선택 >> 0

정말 종료하시겠습니까? (Y/N) >> Y
시스템을 종료합니다.
```

```
메뉴 선택 >> 0

정말 종료하시겠습니까? (Y/N) >> N
[안내] 계속 진행합니다.
```

`Y` / `y` 이외의 모든 입력은 취소로 처리한다.


## 4. 공통 UI 규칙 (전 Phase 적용)

| 규칙 | 값 / 형식 |
|------|-----------|
| 테이블 너비 | 고정 60자 |
| 테이블 구분선 | `─` × 60 |
| 박스 메뉴 내부 너비 | 38자 (양쪽 `║` 포함 시 40자) |
| 박스 메뉴 상단 | `╔═…═╗` |
| 박스 메뉴 제목 구분 | `╠═…═╣` |
| 박스 메뉴 하단 | `╚═…═╝` |
| 오류 메시지 | `[오류] …` |
| 성공 메시지 | `[완료] …` |
| 안내 메시지 | `[안내] …` |
| 뒤로가기 | `0` 입력 → 상위 메뉴 복귀 |
| 빈 입력 처리 | 재입력 요청 또는 취소로 처리 (컨텍스트에 따라 결정) |
| 확인 프롬프트 | `(Y/N) >>` 형식, Y/y만 긍정 |
| 한국어 출력 | `sys.stdout.reconfigure(encoding="utf-8")` 필수 |

## 5. 데이터 흐름

```
main.py
    │
    ▼
AppContext 생성 (모든 repo·service 초기화)
    │
    ▼
main_menu.run(ctx)
    │
    ├─ 시작 배너 출력 (최초 1회)
    │
    └─ 루프:
        ├─ _collect_summary(ctx)
        │     ├─ sample_repo.find_all()            → 등록 시료 수, 총 재고
        │     ├─ order_repo.count_by_status()       → 주문 현황
        │     └─ production_repo.find_by_status(WAITING) → 생산 대기
        │
        ├─ _print_summary(summary)
        ├─ 메인 메뉴 출력
        ├─ 입력 수신
        ├─ "0" → _confirm_exit() → 종료 또는 복귀
        ├─ "1"~"6" → _dispatch() → importlib로 서브메뉴 동적 로드
        └─ 기타 → [오류] 메시지 → 재루프
```

### 집계 로직

```python
samples = sample_repo.find_all()
counts  = order_repo.count_by_status()

등록 시료  = len(samples)
총 재고   = sum(s.stock for s in samples)
전체 주문 = sum(counts[s] for s in [RESERVED, PRODUCING, CONFIRMED, RELEASE])
생산 대기 = len(production_repo.find_by_status(WAITING))
```
