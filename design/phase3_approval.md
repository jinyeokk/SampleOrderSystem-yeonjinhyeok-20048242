# Phase 3 : 주문 승인 / 거절

## 1. 개요

메인 메뉴 **[3] 주문 승인/거절** 선택 시 서브메뉴 없이 RESERVED 주문 목록이 바로 표시된다.  
목록에서 번호를 선택하면 해당 주문의 **승인 또는 거절**을 즉시 처리한다.

| 액션 | 조건 | 결과 상태 |
|------|------|-----------|
| 승인 | 재고 ≥ 주문 수량 | `CONFIRMED` (재고 즉시 차감) |
| 승인 | 재고 < 주문 수량 | `PRODUCING` (생산 라인 자동 등록) |
| 거절 | - | `REJECTED` |

---

```
메인 메뉴 [3]
    │
    ▼
접수 주문 목록 (RESERVED)

    │                      │

    │
    └── 0 입력 ──► 메인 메뉴로
```

## 3. 기능 상세

### 3.1 접수 주문 목록

```
────────────────────────────────────────────────────────────
  접수 주문 목록 (총 3건)
────────────────────────────────────────────────────────────
 NO  주문 번호              고객명              시료명        수량
────────────────────────────────────────────────────────────
  1  ORD-20260508-001    (주)반도체코리아    실리콘 기판     30 ea
  2  ORD-20260508-002    ABC전자             인화인듐 박막   20 ea
  3  ORD-20260508-003    XYZ소재             갈륨비소 웨이퍼 80 ea
────────────────────────────────────────────────────────────
번호 선택 (0: 메인 메뉴) >> 
```

#### 컬럼 구성 (합계 60자)

| 컬럼 | 너비 | 정렬 |
|------|------|------|
| NO | 4 | 우 |
| 주문 번호 | 20 | 좌 |
| 고객명 | 16 (display) | 좌 |
| 시료명 | 14 (display) | 좌 |
| 수량 | 6 | 우 |

#### 표시 규칙

| 항목 | 내용 |
|------|------|
| 정렬 | 접수일시 오름차순 (오래된 순 우선) |
| 목록 없음 | `[안내] 승인 대기 중인 주문이 없습니다.` 후 메인 복귀 |

---

### 3.2 주문 정보 + 승인/거절 선택

번호 선택 시 주문 정보와 함께 현재 재고를 표시한 후 액션을 선택한다.

```
────────────────────────────────────────────────────────────
  주문 정보
────────────────────────────────────────────────────────────
  주문 번호 : ORD-20260508-001
  고객명    : (주)반도체코리아
  시료      : 실리콘 기판 (A-002)
  주문 수량 : 30 ea
  현재 재고 : 45 ea
────────────────────────────────────────────────────────────
  1. 승인
  2. 거절
  0. 취소
선택 >> 
```

---

### 3.3 승인 처리 결과

#### 재고 충분 → CONFIRMED

```
선택 >> 1

[완료] 주문 승인 완료.
────────────────────────────────────────────────────────────
  주문 번호 : ORD-20260508-001
  변경 상태 : RESERVED → CONFIRMED
  재고 변동 : 45 ea → 15 ea
────────────────────────────────────────────────────────────
```

#### 재고 부족 → PRODUCING

```
선택 >> 1

[완료] 주문 승인 완료.
────────────────────────────────────────────────────────────
  주문 번호 : ORD-20260508-002
  변경 상태 : RESERVED → PRODUCING
  생산 등록 : 27 ea  (ceil(20 / 0.75), 수율 75%)
────────────────────────────────────────────────────────────
```

생산 필요량 계산:
```
생산 필요량 = ceil(주문 수량 / 수율)
```

### 3.4 거절 처리 결과

```
────────────────────────────────────────────────────────────
  주문 번호 : ORD-20260508-003
  변경 상태 : RESERVED → REJECTED
────────────────────────────────────────────────────────────
```

### 3.5 오류 케이스

| 오류 | 메시지 |
|------|--------|
| 유효하지 않은 번호 | `[오류] 유효하지 않은 번호입니다.` |

> 승인·거절 대상은 목록에서 선택한 RESERVED 주문이므로 상태 오류는 발생하지 않는다.

---

## 4. 상태 전이

```
RESERVED ──[승인, 재고 충분]──► CONFIRMED  (stock -= quantity)
RESERVED ──[승인, 재고 부족]──► PRODUCING  (ProductionQueue WAITING 등록)
RESERVED ──[거절]─────────────► REJECTED
```

---

## 5. 데이터 흐름

```
메인 메뉴 [3] 선택
    │
    ▼
approval_menu.run(ctx)
    │
    ├─ order_service.get_all(RESERVED) → 목록 표시
    │
    ├─ 번호 선택 → 해당 Order 조회
    │              sample_service.get(sample_id) → 시료명·재고 표시
    │
    ├─ 1.승인 → order_service.approve(order_id)
    │            ├─ [재고 충분] sample.stock -= quantity / status → CONFIRMED
    │            └─ [재고 부족] ProductionQueue 생성 / status → PRODUCING
    │
    └─ 2.거절 → order_service.reject(order_id)
                 status → REJECTED
```
