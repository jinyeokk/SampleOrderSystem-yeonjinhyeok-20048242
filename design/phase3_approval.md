# Phase 3 : 주문 승인 / 거절

## 1. 개요

메인 메뉴 **[3] 주문 승인/거절** 선택 시 서브메뉴 없이 RESERVED 주문 목록이 바로 표시된다.  
목록에서 번호를 선택하면 해당 주문의 **승인 또는 거절**을 즉시 처리한다.

| 액션 | 조건 | 결과 상태 |
|------|------|-----------|
| 승인 | 재고 ≥ 주문 수량 | `CONFIRMED` (재고 즉시 차감) |
| 승인 | 재고 < 주문 수량 | `PRODUCING` (생산 라인 자동 등록) |
| 거절 | - | `REJECTED` |


## 3. 기능 상세

### 3.1 승인 대기 중인 예약 목록

```
────────────────────────────────────────────────────────────
  승인 대기 중인 예약 목록 (총 3건)
────────────────────────────────────────────────────────────
 NO  주문 번호              고객명              시료명        수량
────────────────────────────────────────────────────────────
  1  ORD-20260508-001    (주)반도체코리아    실리콘 기판     30 ea
  2  ORD-20260508-002    ABC전자             인화인듐 박막   20 ea
  3  ORD-20260508-003    XYZ소재             갈륨비소 웨이퍼 80 ea
────────────────────────────────────────────────────────────
승인할 번호 (0: 메인 메뉴) >> 
```

#### 컬럼 구성 (합계 60자)

| 컬럼 | 너비 | 정렬 |
|------|------|------|
| NO | 4 | 우 |
| 주문 번호 | 20 | 좌 |
| 고객명 | 16 (display) | 좌 |
| 시료명 | 14 (display) | 좌 |
| 수량 | 6 | 우 |

#### 표시 규칙

| 항목 | 내용 |
|------|------|
| 정렬 | 접수일시 오름차순 (오래된 순 우선) |
| 목록 없음 | `[안내] 승인 대기 중인 주문이 없습니다.` 후 메인 복귀 |

---

### 3.2 재고 확인 중

번호 선택 시 재고를 확인하고 승인/주문거절을 선택한다.

#### 재고 충분 (현재 재고 ≥ 주문 수량)

```
────────────────────────────────────────────────────────────
  재고 확인 중
────────────────────────────────────────────────────────────
  주문 번호 : ORD-20260508-001
  고객명    : (주)반도체코리아
  시료      : 실리콘 기판 (A-002)
  주문 수량 : 30 ea
  현재 재고 : 45 ea
────────────────────────────────────────────────────────────
  1. 승인
  2. 주문거절
  0. 취소
선택 >> 
```

#### 재고 부족 (현재 재고 < 주문 수량)

부족분을 표시하고 노란 글씨로 생산 안내를 출력한다.

```
────────────────────────────────────────────────────────────
  재고 확인 중
────────────────────────────────────────────────────────────
  주문 번호 : ORD-20260508-002
  고객명    : ABC전자
  시료      : 인화인듐 박막 (B-001)
  주문 수량 : 20 ea
  현재 재고 :  0 ea
  부족분    : 20 ea  ← 이 수량만 생산
────────────────────────────────────────────────────────────
⚠ 재고 부족. 부족분 20ea 승인하시겠습니까? (실 생산량 27ea / 3,240min)   [노란색]
────────────────────────────────────────────────────────────
  1. 승인
  2. 주문거절
  0. 취소
선택 >> 
```

| 항목 | 계산식 |
|------|--------|
| 부족분 | 주문 수량 − 현재 재고 |
| 실 생산량 | `ceil(부족분 / 수율)` |
| 예상 시간 | `실 생산량 × 평균 생산시간 (min/ea)` |

> 노란색 출력: ANSI `\033[33m … \033[0m`

---

### 3.3 승인 처리 결과

#### 재고 충분 → CONFIRMED

```
선택 >> 1

[완료] 주문 승인 완료.
────────────────────────────────────────────────────────────
  주문 번호 : ORD-20260508-001
  변경 상태 : RESERVED → CONFIRMED
  재고 변동 : 45 ea → 15 ea
────────────────────────────────────────────────────────────
```

#### 재고 부족 → PRODUCING

```
선택 >> 1

[완료] 주문 승인 완료.
────────────────────────────────────────────────────────────
  주문 번호 : ORD-20260508-002
  변경 상태 : RESERVED → PRODUCING
  생산 등록 : 27 ea  (ceil(부족분 20 / 수율 0.75))
────────────────────────────────────────────────────────────
```

실 생산량 계산:
```
부족분    = 주문 수량 - 현재 재고
실 생산량 = ceil(부족분 / 수율)
예상 시간 = 실 생산량 × 평균 생산시간 (min/ea)
```

### 3.5 오류 케이스

| 오류 | 메시지 |
|------|--------|
| 유효하지 않은 번호 | `[오류] 유효하지 않은 번호입니다.` |

> 승인·거절 대상은 목록에서 선택한 RESERVED 주문이므로 상태 오류는 발생하지 않는다.

---

## 4. 상태 전이

```
RESERVED ──[승인, 재고 충분]──► CONFIRMED  (stock -= quantity)
RESERVED ──[승인, 재고 부족]──► PRODUCING  (부족분 기준 ProductionQueue WAITING 등록)
RESERVED ──[주문거절]──────────► REJECTED
```

---

## 5. 데이터 흐름

```
메인 메뉴 [3] 선택
    │
    ▼
approval_menu.run(ctx)
    │
    ├─ order_service.get_all(RESERVED) → 목록 표시
    │
    ├─ 승인할 번호 선택 → 해당 Order 조회
    │              sample_service.get(sample_id) → 시료명·재고 조회
    │              부족분 = order.quantity - sample.stock (< 0이면 충분)
    │              [부족] 실 생산량 = ceil(부족분 / yield_rate), 예상 시간 계산
    │              노란색 경고 표시
    │
    ├─ 1.승인 → order_service.approve(order_id)
    │            ├─ [재고 충분] sample.stock -= quantity / status → CONFIRMED
    │            └─ [재고 부족] ProductionQueue(required_qty=ceil(부족분/yield_rate))
    │                           status → PRODUCING
    │
    └─ 2.주문거절 → order_service.reject(order_id)
                    status → REJECTED
```
