## 2. 기능 상세

### 2.1 승인 대기 주문 목록

- `RESERVED` 상태 주문만 표시

```
──────────────────────────────────────────────────────────────────────
  승인 대기 주문 목록 (총 3건)
──────────────────────────────────────────────────────────────────────
 주문 ID              고객명              시료명         수량  현재재고  재고상태
──────────────────────────────────────────────────────────────────────
 ORD-20260508-001    (주)반도체코리아    실리콘 기판      30     45      충분
 ORD-20260508-002    ABC전자             인화인듐 박막    20      0      부족 ⚠
 ORD-20260508-003    XYZ소재             갈륨비소 웨이퍼  80     50      부족 ⚠
──────────────────────────────────────────────────────────────────────
```

#### 재고 상태 판단

| 조건 | 표시 |
|------|------|
| 재고 ≥ 주문 수량 | `충분` |
| 재고 < 주문 수량 | `부족 ⚠` |

### 2.2 주문 승인 처리

#### 입력 흐름

```
──────────────────────────────────────
  주문 승인 처리
──────────────────────────────────────
주문 ID 입력 >> ORD-20260508-001

──────────────────────────────────────
  주문 정보
──────────────────────────────────────
  주문 ID    : ORD-20260508-001
  고객명     : (주)반도체코리아
  시료       : A-002 실리콘 기판
  주문 수량  : 30 개
  현재 재고  : 45 개
  → 재고 충분 → 승인 시 CONFIRMED 처리
──────────────────────────────────────
승인하시겠습니까? (Y/N) >> Y

[완료] 주문 ORD-20260508-001 승인 완료.
       상태: RESERVED → CONFIRMED
       재고 차감: 45개 → 15개
```

#### 재고 부족 케이스

```
  주문 ID    : ORD-20260508-002
  고객명     : ABC전자
  시료       : B-001 인화인듐 박막
  주문 수량  : 20 개
  현재 재고  :  0 개
  → 재고 부족 → 승인 시 PRODUCING 처리

  [생산 정보 미리보기]
  수율         : 75%
  실제 생산량  : 27개 (ceil(20 / 0.75))
  예상 소요시간: 3,240분 (27개 × 120분/개)
──────────────────────────────────────
승인하시겠습니까? (Y/N) >> Y

[완료] 주문 ORD-20260508-002 승인 완료.
       상태: RESERVED → PRODUCING
       생산 큐 등록: 27개 생산 예정
```

#### 승인 처리 로직

```
승인 처리 흐름:
    1. 주문 상태 확인 (RESERVED 아니면 오류)
    2. 재고 확인
    3-A. 재고 >= 주문수량:
         - 재고 차감
         - 상태 → CONFIRMED
    3-B. 재고 < 주문수량:
         - required = ceil(주문수량 / yield_rate)
         - 생산 큐에 등록 (required_qty, status=WAITING)
         - 상태 → PRODUCING
```

#### 오류 케이스

| 오류 | 메시지 |
|------|--------|
| 존재하지 않는 주문 ID | `[오류] 해당 주문을 찾을 수 없습니다.` |
| RESERVED 아닌 주문 | `[오류] 승인 가능한 상태가 아닙니다. (현재: CONFIRMED)` |

### 2.3 주문 거절 처리

#### 입력 흐름

```
──────────────────────────────────────
  주문 거절 처리
──────────────────────────────────────
주문 ID 입력 >> ORD-20260508-003

──────────────────────────────────────
  주문 정보
──────────────────────────────────────
  주문 ID    : ORD-20260508-003
  고객명     : XYZ소재
  시료       : A-001 갈륨비소 웨이퍼
  주문 수량  : 80 개
  현재 재고  : 50 개
──────────────────────────────────────
거절 사유 입력 >> 해당 시료 생산 라인 정비 중으로 수주 불가

거절 처리하시겠습니까? (Y/N) >> Y

[완료] 주문 ORD-20260508-003 거절 처리.
       상태: RESERVED → REJECTED
```

#### 거절 규칙

| 항목 | 내용 |
|------|------|
| 거절 사유 | 필수 입력 (빈 값 불가) |
| 거절 가능 상태 | `RESERVED` 만 허용 |
| 거절 후 | 모니터링 집계에서 제외 |

## 3. 데이터 흐름

```
사용자 입력 (주문 ID)
    │
    ▼
approval_menu.py (화면 입출력)
    │
    ▼
order_service.approve(order_id)
    ├─ order_repo.get(order_id)         → 주문 조회
    ├─ sample_repo.get(sample_id)       → 시료/재고 조회
    ├─ [재고 충분] sample_repo.update_stock(-qty)
    │                order_repo.update_status(CONFIRMED)
    └─ [재고 부족] production_repo.enqueue(order)
                  order_repo.update_status(PRODUCING)
```
