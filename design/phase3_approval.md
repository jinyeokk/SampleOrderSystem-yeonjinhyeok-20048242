# Phase 3 : 주문 승인 / 거절

## 1. 개요

RESERVED 상태의 접수된 주문을 확인하고, 주문별로 **승인** 또는 **거절**을 처리한다.

| 액션 | 조건 | 결과 상태 |
|------|------|-----------|
| 승인 | 재고 ≥ 주문 수량 | `CONFIRMED` (재고 즉시 차감) |
| 승인 | 재고 < 주문 수량 | `PRODUCING` (생산 라인 자동 등록) |
| 거절 | - | `REJECTED` |

## 3. 기능 상세

### 3.1 접수 주문 목록

`RESERVED` 상태 주문만 표시한다.

```
────────────────────────────────────────────────────────────
  접수 주문 목록 (총 3건)
────────────────────────────────────────────────────────────
 주문 번호             고객명              시료명          수량  현재 재고
────────────────────────────────────────────────────────────
 ORD-20260508-001    (주)반도체코리아    실리콘 기판       30    45 ea
 ORD-20260508-002    ABC전자             인화인듐 박막     20     0 ea ⚠
 ORD-20260508-003    XYZ소재             갈륨비소 웨이퍼   80    50 ea
────────────────────────────────────────────────────────────
```

#### 컬럼 구성 (합계 60자)

| 컬럼 | 너비 | 정렬 |
|------|------|------|
| 주문 번호 | 20 | 좌 |
| 고객명 | 16 (display) | 좌 |
| 시료명 | 14 (display) | 좌 |
| 수량 | 4 | 우 |
| 현재 재고 | 6 | 우 |

#### 표시 규칙

| 항목 | 표시 방식 |
|------|-----------|
| 현재 재고 | `{값} ea`, 0이면 `0 ea ⚠` |
| 정렬 | 접수일시 오름차순 (오래된 순) |
| 목록 없음 | `[안내] 승인 대기 중인 주문이 없습니다.` |

---

### 3.2 주문 승인

주문 번호를 입력하면 재고 상황을 확인하고 자동으로 상태를 전환한다.

#### 입력 흐름 — 재고 충분 (CONFIRMED 전환)

```
────────────────────────────────────────────────────────────
  주문 승인
────────────────────────────────────────────────────────────
주문 번호 >> ORD-20260508-001

────────────────────────────────────────────────────────────
  주문 정보
────────────────────────────────────────────────────────────
  주문 번호 : ORD-20260508-001
  고객명    : (주)반도체코리아
  시료      : 실리콘 기판 (A-002)
  주문 수량 : 30 ea
  현재 재고 : 45 ea
────────────────────────────────────────────────────────────
  Y. 승인
  N. 취소
선택 >> Y

[완료] 주문 승인 완료.
────────────────────────────────────────────────────────────
  주문 번호 : ORD-20260508-001
  변경 상태 : RESERVED → CONFIRMED
  재고 변동 : 45 ea → 15 ea
────────────────────────────────────────────────────────────
```

#### 입력 흐름 — 재고 부족 (PRODUCING 전환)

```
────────────────────────────────────────────────────────────
  주문 승인
────────────────────────────────────────────────────────────
주문 번호 >> ORD-20260508-002

────────────────────────────────────────────────────────────
  주문 정보
────────────────────────────────────────────────────────────
  주문 번호 : ORD-20260508-002
  고객명    : ABC전자
  시료      : 인화인듐 박막 (B-001)
  주문 수량 : 20 ea
  현재 재고 : 0 ea ⚠
────────────────────────────────────────────────────────────
  Y. 승인
  N. 취소
선택 >> Y

[완료] 주문 승인 완료.
────────────────────────────────────────────────────────────
  주문 번호 : ORD-20260508-002
  변경 상태 : RESERVED → PRODUCING
  생산 등록 : 27 ea  (ceil(20 / 0.75), 수율 75%)
────────────────────────────────────────────────────────────
```

#### 생산 등록 계산식

```
생산 필요량 = ceil(주문 수량 / 수율)
예) ceil(20 / 0.75) = 27 ea
```

#### 오류 케이스

| 오류 | 메시지 |
|------|--------|
| 존재하지 않는 주문 번호 | `[오류] 해당 주문을 찾을 수 없습니다.` |
| RESERVED 아닌 주문 | `[오류] 승인 가능한 주문이 아닙니다. (현재 상태: CONFIRMED)` |

## 4. 상태 전이 규칙

```
RESERVED ──[승인, 재고 충분]──► CONFIRMED
RESERVED ──[승인, 재고 부족]──► PRODUCING  (생산 큐 WAITING 등록)
RESERVED ──[거절]─────────────► REJECTED
```

- 승인·거절 가능 상태: `RESERVED`만 허용
- `CONFIRMED` 전환 시: `stock -= quantity`
- `PRODUCING` 전환 시: `ProductionQueue(required_qty=ceil(quantity / yield_rate))` 생성

---

## 5. 데이터 흐름

```
approval_menu.py
    │
    ├─ 접수 주문 목록
    │   └─ order_service.get_all(RESERVED)
    │      sample_service.get(sample_id) → 시료명·재고 조회
    │
    ├─ 주문 승인
    │   └─ order_service.approve(order_id)
    │       ├─ [재고 충분] sample_repo.update_stock(-qty)
    │       │              order.status → CONFIRMED
    │       └─ [재고 부족] production_repo.save(queue)
    │                      order.status → PRODUCING
    │
    └─ 주문 거절
        └─ order_service.reject(order_id)
               order.status → REJECTED
```
