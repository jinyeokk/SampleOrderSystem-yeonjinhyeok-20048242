# Phase 5 : 생산라인

## 1. 개요

생산라인에 등록된 주문의 생산 현황과 대기 큐를 관리한다.  
부족분을 기준으로 수율 및 오차를 반영한 **실 생산량**을 계산하여 생산하며,  
생산 완료 시 주문 상태를 `PRODUCING → CONFIRMED`으로 전환한다.

---

## 2. 생산량 계산 공식

```
부족분    = 주문 수량 - 승인 시점 재고
실 생산량 = ceil(부족분 / (수율 × 0.9))   ← 오차 10% 반영
총 생산시간 = 평균 생산시간 × 실 생산량
```

### 계산 예시

| 항목 | 값 |
|------|-----|
| 주문 수량 | 20 ea |
| 승인 시점 재고 | 0 ea |
| 부족분 | 20 ea |
| 수율 | 75% |
| 실 생산량 | `ceil(20 / (0.75 × 0.9))` = `ceil(29.63)` = **30 ea** |
| 평균 생산시간 | 120 min/ea |
| 총 생산시간 | 120 × 30 = **3,600 min** |

> 오차 계수 `0.9`는 수율 불확실성을 10% 추가 반영하여 생산 부족을 방지한다.

## 4. 기능 상세

### 4.1 생산 현황

현재 `PRODUCING` 상태인 주문과 연결된 생산 큐 정보를 표시한다.

```
────────────────────────────────────────────────────────────
  생산 현황 (총 2건)
────────────────────────────────────────────────────────────
 주문 번호             고객명          시료명        주문량   실 생산량   총 생산시간
────────────────────────────────────────────────────────────
 ORD-20260508-002    ABC전자         인화인듐 박막    20 ea    30 ea    3,600 min
 ORD-20260508-004    XYZ소재         실리콘 기판      50 ea    62 ea    1,860 min
────────────────────────────────────────────────────────────
```

| 컬럼 | 설명 |
|------|------|
| 주문량 | 고객 주문 수량 |
| 실 생산량 | `ceil(부족분 / (수율 × 0.9))` |
| 총 생산시간 | `평균 생산시간 × 실 생산량` |

#### 생산 없음

```
[안내] 현재 생산 중인 주문이 없습니다.
```

---

### 4.2 대기 주문

생산 큐에서 `WAITING` 상태인 항목을 **FIFO(선입선출)** 순으로 표시한다.

```
────────────────────────────────────────────────────────────
  대기 주문 (총 3건) — FIFO
────────────────────────────────────────────────────────────
 순번  주문 번호             시료명          실 생산량   큐 등록일시
────────────────────────────────────────────────────────────
   1   ORD-20260508-005    갈륨비소 웨이퍼    45 ea    2026-05-08 10:11
   2   ORD-20260508-006    실리콘 기판         62 ea    2026-05-08 11:33
   3   ORD-20260508-007    인화인듐 박막       30 ea    2026-05-08 14:02
────────────────────────────────────────────────────────────
```

| 항목 | 내용 |
|------|------|
| 정렬 기준 | 큐 등록일시 오름차순 (FIFO) |
| 대기 없음 | `[안내] 대기 중인 생산 주문이 없습니다.` |

---

### 4.3 생산 완료 처리

`PRODUCING` 상태의 주문을 선택해 생산 완료를 처리한다.  
완료 시 재고를 보정하고 주문 상태를 `CONFIRMED`으로 전환한다.

#### 입력 흐름

```
────────────────────────────────────────────────────────────
  생산 완료 처리
────────────────────────────────────────────────────────────
 NO  주문 번호             고객명          시료명        주문량   실 생산량
────────────────────────────────────────────────────────────
  1  ORD-20260508-002    ABC전자         인화인듐 박막    20 ea    30 ea
  2  ORD-20260508-004    XYZ소재         실리콘 기판      50 ea    62 ea
────────────────────────────────────────────────────────────
완료 처리할 번호 (0: 취소) >> 1

────────────────────────────────────────────────────────────
  생산 완료 확인
────────────────────────────────────────────────────────────
  주문 번호 : ORD-20260508-002
  고객명    : ABC전자
  시료      : 인화인듐 박막 (B-001)
  주문 수량 : 20 ea
  실 생산량 : 30 ea
  재고 변동 : 0 ea + 30 ea 생산 → 20 ea 출고 차감 → 잔여 10 ea
────────────────────────────────────────────────────────────
  Y. 완료 처리
  N. 취소
선택 >> Y

[완료] 생산 완료 처리됨.
────────────────────────────────────────────────────────────
  주문 번호 : ORD-20260508-002
  변경 상태 : PRODUCING → CONFIRMED
  최종 재고 : 10 ea
────────────────────────────────────────────────────────────
```

#### 생산 완료 재고 처리

```
재고 변동 = 실 생산량 - 주문 수량
최종 재고 = 기존 재고 + 재고 변동
         = 기존 재고 + 실 생산량 - 주문 수량
```

#### 오류 케이스

| 오류 | 메시지 |
|------|--------|
| 유효하지 않은 번호 | `[오류] 유효하지 않은 번호입니다.` |
| 생산 중 주문 없음 | `[안내] 생산 완료 처리할 주문이 없습니다.` |

---

## 5. 상태 전이

```
PRODUCING ──[생산 완료]──► CONFIRMED
              ↓
        재고 += (실 생산량 - 주문 수량)
        ProductionQueue.status → DONE
```

---

## 6. 데이터 흐름

```
approval_menu (Phase 3)
    └─ approve() → PRODUCING 전환 시
          required_qty = ceil(부족분 / (수율 × 0.9))
          ProductionQueue(required_qty, status=WAITING)

production_menu.run(ctx)
    │
    ├─ 1. 생산 현황
    │   └─ order_service.get_all(PRODUCING)
    │      production_repo.find_by_order(order_id) → 큐 정보 조회
    │
    ├─ 2. 대기 주문
    │   └─ production_repo.find_by_status(WAITING)
    │      .sort(queued_at ASC)  ← FIFO
    │
    └─ 3. 생산 완료 처리
        └─ production_service.complete_production(order_id)
            ├─ sample.stock += required_qty - order.quantity
            ├─ ProductionQueue.status → DONE
            └─ order.status → CONFIRMED
```
