# Phase 5 : 생산라인

## 1. 개요

메인 메뉴 **[5] 생산라인** 선택 시 서브메뉴 없이 생산라인 대시보드가 바로 표시된다.  
단일 생산라인의 **현재 상태(RUNNING / STOP)**, **현재 처리 중인 건**, **대기 주문**을 한 화면에서 파악한다.

| 항목 | 조건 |
|------|------|
| RUNNING | PRODUCING 상태 주문이 1건 이상 존재 |
| STOP | PRODUCING 상태 주문이 없음 |

---

## 2. 생산량 계산 공식

```
부족분     = 주문 수량 - 승인 시점 재고
실 생산량  = ceil(부족분 / (수율 × 0.9))   ← 오차 10% 안전 반영
총 생산시간 = 평균 생산시간 × 실 생산량
```

**예시** — 부족분 20 ea, 수율 75%, 평균 생산시간 120 min/ea

```
실 생산량  = ceil(20 / (0.75 × 0.9)) = ceil(29.63) = 30 ea
총 생산시간 = 120 × 30 = 3,600 min
```

---

## 3. 화면 흐름

```
메인 메뉴 [5]
    │
    ▼
생산라인 대시보드 출력  ← 서브메뉴 없음
    │  ┌─────────────────────────────────┐
    │  │ 생산라인 상태 : ■ RUNNING / □ STOP  │
    │  │ 현재 처리 중 (FIFO 1번)          │
    │  │ 대기 주문 목록 (FIFO 순)         │
    │  └─────────────────────────────────┘
    │
    └── 0. 메인 메뉴
```

#### 진행률 표시 규칙

| 항목 | 내용 |
|------|------|
| 바 너비 | 20칸 (`▓` 완료 / `░` 미완료) |
| 진행 수식 | `filled = int(produced_qty / required_qty * 20)` |
| 초기값 | 생산 시작 직후 0 / 30 ea → `[░░░░░░░░░░░░░░░░░░░░]` |

> 진행률은 생산이 완료될 때까지 0%로 표시된다 (시뮬레이션 특성).

## 5. 현재 처리 중 / 대기 주문 결정 기준

PRODUCING 상태 주문은 **생산 큐 등록일시(queued_at) 오름차순(FIFO)**으로 정렬한다.

| 구분 | 대상 |
|------|------|
| 현재 처리 중 | FIFO 첫 번째 PRODUCING 주문 |
| 대기 주문 | FIFO 두 번째 이후 PRODUCING 주문 (큐 등록일시 오름차순) |

---

## 6. 상태 전이

```
PRODUCING ──[생산 완료]──► CONFIRMED
              ↓
        stock += (실 생산량 - 주문 수량)
        ProductionQueue.status → DONE
```
