# Phase 4 : 모니터링

## 1. 개요

담당자가 현재 시스템 상태를 한눈에 파악할 수 있도록 **주문량**과 **재고량**을 메뉴로 선택해 확인한다.

| 메뉴 | 내용 |
|------|------|
| 1. 주문량 확인 | 상태별 주문 건수 (REJECTED 제외) |
| 2. 재고량 확인 | 시료별 현재 재고 + 예약 수량 + 재고 상태 |

## 3. 기능 상세

### 3.1 주문량 확인

상태별 주문 건수를 표시한다. `REJECTED`는 유효한 주문이 아니므로 제외한다.

```
────────────────────────────────────────────────────────────
  주문량 확인                          2026-05-08 15:30:00
────────────────────────────────────────────────────────────
  RESERVED    PRODUCING    CONFIRMED    RELEASE
  ──────────  ──────────   ──────────   ───────
      3건          2건          5건        12건
────────────────────────────────────────────────────────────
```

| 항목 | 내용 |
|------|------|
| 표시 상태 | RESERVED / PRODUCING / CONFIRMED / RELEASE |
| 제외 상태 | REJECTED |
| 우측 상단 | 현재 시각 (`YYYY-MM-DD HH:MM:SS`) |

---

### 3.2 재고량 확인

각 시료별 현재 재고와 예약 수량을 비교하여 재고 상태를 표시한다.

```
────────────────────────────────────────────────────────────
  재고량 확인
────────────────────────────────────────────────────────────
 시료 ID    시료명              현재 재고    예약 수량    상태
────────────────────────────────────────────────────────────
 A-001     갈륨비소 웨이퍼       120 ea       30 ea      여유
 A-002     실리콘 기판            15 ea       30 ea      부족
 B-001     인화인듐 박막           0 ea        0 ea      고갈
────────────────────────────────────────────────────────────
```

#### 컬럼 구성 (합계 60자)

| 컬럼 | 너비 | 정렬 |
|------|------|------|
| 시료 ID | 10 | 좌 |
| 시료명 | 18 (display) | 좌 |
| 현재 재고 | 10 | 우 |
| 예약 수량 | 10 | 우 |
| 상태 | 6 | 우 |

#### 재고 상태 판단 기준

| 상태 | 조건 |
|------|------|
| 고갈 | `현재 재고 == 0` |
| 부족 | `현재 재고 > 0` AND `현재 재고 < 예약 수량` |
| 여유 | `현재 재고 >= 예약 수량` (재고 > 0) |

> **예약 수량**: 해당 시료의 `RESERVED` 상태 주문 수량 합계

## 4. 데이터 집계 로직

```
주문량 확인:
  counts = order_repo.count_by_status()
  표시: RESERVED / PRODUCING / CONFIRMED / RELEASE 건수

재고량 확인:
  for each sample in sample_repo.find_all():
    현재 재고 = sample.stock
    예약 수량 = sum(o.quantity
                   for o in orders
                   if o.sample_id == sample.sample_id
                   and o.status == RESERVED)
    상태 = _stock_label(현재 재고, 예약 수량)

_stock_label(stock, reserved_qty):
  if stock == 0            → "고갈"
  if stock < reserved_qty  → "부족"
  else                     → "여유"
```

