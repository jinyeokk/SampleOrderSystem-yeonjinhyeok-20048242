# 반도체 시료 생산주문관리 시스템 PRD

## 1. 개요

반도체 시료의 주문 접수부터 생산, 출고까지 전 과정을 관리하는 웹 기반 시스템이다.  
생산 라인 담당자와 주문 관리자가 실시간으로 주문 상태를 추적하고, 재고 및 생산 현황을 모니터링할 수 있도록 한다.

---

## 2. 주문 상태 흐름

```
RESERVED → CONFIRMED → RELEASE
         ↘ REJECTED
         ↘ PRODUCING → CONFIRMED → RELEASE
```

| 상태 | 의미 | 모니터링 포함 여부 |
|------|------|--------------------|
| `RESERVED` | 주문 접수 (승인 대기 중) | ✅ |
| `REJECTED` | 주문 거절 | ❌ (제외) |
| `PRODUCING` | 주문 승인 완료 · 재고 부족으로 생산 중 | ✅ |
| `CONFIRMED` | 주문 승인 완료 · 출고 대기 중 | ✅ |
| `RELEASE` | 출고 완료 | ✅ |

### 상태 전이 규칙

- `RESERVED` → `CONFIRMED` : 승인 시 재고가 충분한 경우
- `RESERVED` → `PRODUCING` : 승인 시 재고가 부족한 경우
- `RESERVED` → `REJECTED` : 거절 처리
- `PRODUCING` → `CONFIRMED` : 생산 완료 후 재고 확보
- `CONFIRMED` → `RELEASE` : 출고 처리 실행

## 3. 메뉴 구성 및 기능 요구사항

### 3.1 시료 관리

**목적** : 시스템에 등록된 반도체 시료를 관리한다.

| 기능 | 설명 |
|------|------|
| 시료 등록 | 시료명, 규격, 초기 재고 수량 등 기본 정보 입력 후 등록 |
| 목록 조회 | 등록된 전체 시료 목록을 테이블로 표시 (이름, 재고 수량, 등록일 포함) |
| 이름 검색 | 시료명 키워드로 필터링하여 빠르게 시료 검색 |

**시료 데이터 필드**

| 필드 | 타입 | 설명 |
|------|------|------|
| `id` | UUID | 고유 식별자 |
| `name` | string | 시료명 |
| `spec` | string | 규격 / 사양 |
| `stock` | integer | 현재 재고 수량 |
| `created_at` | datetime | 등록일시 |

---

### 3.2 주문 (접수 / 승인 / 거절)

**목적** : 고객 주문을 접수하고, 생산 라인 담당자가 승인 또는 거절 처리한다.

#### 3.2.1 주문 접수

| 기능 | 설명 |
|------|------|
| 주문 등록 | 고객명, 시료 선택, 수량 입력 후 주문 접수 → 상태 `RESERVED` |
| 주문 목록 | 전체 주문 목록 조회 (상태별 필터링 가능) |
| 주문 상세 | 주문 단건 상세 정보 확인 |

#### 3.2.2 승인 처리

| 조건 | 동작 |
|------|------|
| 재고 ≥ 주문 수량 | 재고 차감 후 상태를 `CONFIRMED`로 변경 |
| 재고 < 주문 수량 | 생산 큐에 등록 후 상태를 `PRODUCING`으로 변경 |

#### 3.2.3 거절 처리

- 담당자가 거절 사유를 입력하고 주문을 `REJECTED` 상태로 변경
- `REJECTED` 주문은 모니터링 집계에서 제외

**주문 데이터 필드**

| 필드 | 타입 | 설명 |
|------|------|------|
| `id` | UUID | 고유 식별자 |
| `customer_name` | string | 고객명 |
| `sample_id` | UUID | 시료 FK |
| `quantity` | integer | 주문 수량 |
| `status` | enum | `RESERVED` \| `REJECTED` \| `PRODUCING` \| `CONFIRMED` \| `RELEASE` |
| `reject_reason` | string? | 거절 사유 (거절 시 필수) |
| `created_at` | datetime | 주문 접수일시 |
| `updated_at` | datetime | 최종 상태 변경일시 |

---

### 3.3 모니터링

**목적** : 주문 현황과 시료별 재고를 한눈에 파악한다.

| 섹션 | 표시 내용 |
|------|-----------|
| 상태별 주문 수 | `RESERVED` / `PRODUCING` / `CONFIRMED` / `RELEASE` 건수 요약 카드 (`REJECTED` 제외) |
| 시료별 재고 현황 | 시료명, 현재 재고, 생산 중 수량, 대기 주문 수량을 테이블로 표시 |
| 최근 주문 활동 | 최근 N건의 상태 변경 이력 스트림 |

---

### 3.4 출고 처리

**목적** : `CONFIRMED` 상태 주문을 실제 출고 처리한다.

| 기능 | 설명 |
|------|------|
| 출고 대기 목록 | 상태가 `CONFIRMED`인 주문 목록 표시 |
| 출고 실행 | 개별 주문 선택 후 출고 확정 → 상태를 `RELEASE`로 변경, 출고일시 기록 |
| 일괄 출고 | 선택한 다수 주문 동시 출고 처리 |

---

### 3.5 생산 라인

**목적** : 현재 생산 중인 시료와 대기 중인 생산 큐를 관리한다.

| 기능 | 설명 |
|------|------|
| 생산 중 목록 | 상태가 `PRODUCING`인 주문 목록 (시료명, 수량, 접수일 표시) |
| 생산 큐 | 대기 순서에 따라 정렬된 생산 예정 주문 목록 |
| 생산 완료 처리 | 생산 완료 시 재고 추가 후 해당 주문을 `CONFIRMED`로 전환 |

---

## 4. 비기능 요구사항

| 항목 | 요건 |
|------|------|
| 플랫폼 | 웹 브라우저 (데스크탑 우선) |
| 응답 속도 | 주요 조회 화면 2초 이내 응답 |
| 데이터 정합성 | 재고 차감 및 상태 전이는 원자적으로 처리 (동시성 충돌 방지) |
| 사용성 | 상태 전이 실수를 방지하기 위한 확인 다이얼로그 제공 |
